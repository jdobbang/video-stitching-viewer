# Soccer Tactics Analyzer — 기능 고도화 기획서

---

## 1. 현재 기능 현황 (AS-IS)

### 핵심 모듈

| 모듈 | 기능 | 완성도 |
|------|------|--------|
| 파노라마 뷰어 | Three.js 기반 360° 영상 재생, FOV/회전 조절, TPS 시점 | ✅ 완성 |
| 필드 캘리브레이션 | 미니맵 포인트 ↔ 화면 좌표 매핑, TPS 보간, 호모그래피 | ✅ 완성 |
| 전술 모드 | 화면 클릭 → 필드 좌표 변환, 미니맵에 포인트 표시 | ⚠️ 기초 수준 |
| 미니맵 | 필드 라인 렌더링, 캘리브레이션 포인트, 줄무늬 시각화 | ✅ 완성 |
| 저장/불러오기 | 캘리브레이션 데이터 JSON export/import | ✅ 완성 |

### 현재 한계점

- 전술 모드가 "포인트 찍기"에 그침 — 선수 구분, 이동 경로, 포메이션 등 분석 불가
- 시간축 연동 없음 — 특정 시점의 전술 스냅샷만 가능, 시간 흐름에 따른 변화 추적 불가
- 단일 사용자 로컬 환경 — 공유·협업 불가
- 출력물 없음 — 분석 결과를 리포트나 이미지로 내보내기 불가

## 2. 고도화 로드맵

### Phase 1 — 선수 & 볼 트래킹 (핵심 기반)

**목표:** 포인트를 "선수"로 승격시켜 개별 식별·추적 가능하게 만든다.

#### 1-1. 선수 객체 모델
- 포인트 → `Player` 객체로 확장: `{ id, name, number, team, position, color, fieldX, fieldY, timestamp }`
- 팀 구분 (홈/어웨이) + 팀별 컬러 프리셋
- 등번호·이름 라벨 표시 (미니맵 + 오버레이)
- GK, DF, MF, FW 등 포지션 태그

#### 1-2. 볼 객체
- 별도 볼 마커 (구별되는 아이콘/색상)
- 볼 소유 선수 자동 연결 (가장 가까운 선수 기준)

#### 1-3. 타임라인 기반 키프레임 시스템
- 영상 타임코드 ↔ 선수 위치를 키프레임으로 저장
- 타임라인 UI: 하단 시크바에 키프레임 마커 표시
- 키프레임 간 위치 보간 (선형/스플라인) → 중간 프레임 자동 생성
- 키프레임 구간 재생 시 미니맵에서 선수 이동 애니메이션

#### 1-4. 드래그 기반 위치 편집
- 미니맵 및 파노라마 화면 위에서 선수 마커를 드래그해 위치 수정
- Undo/Redo 스택 (Ctrl+Z / Ctrl+Y)

---

### Phase 2 — 전술 드로잉 & 포메이션

**목표:** 코치가 쓰는 전술 보드 수준의 드로잉 도구를 탑재한다.

#### 2-1. 드로잉 레이어
- 화살표(이동 경로), 직선, 곡선(베지어), 자유곡선 도구
- 선 스타일: 실선(패스), 점선(런), 물결선(드리블)
- 영역 하이라이트: 사각형, 원, 자유 영역 (공간 강조용)
- 텍스트 주석 추가

#### 2-2. 포메이션 템플릿
- 표준 포메이션 프리셋: 4-3-3, 4-4-2, 3-5-2, 4-2-3-1 등
- 원클릭으로 11명 선수 자동 배치
- 커스텀 포메이션 저장/불러오기
- 공격 시 / 수비 시 포메이션 전환 비교 뷰

#### 2-3. 전술 패턴 라이브러리
- 빌드업, 프레싱, 역습, 세트피스 등 전술 패턴 템플릿
- 선수 이동 화살표가 포함된 애니메이션 프리셋
- 사용자 정의 패턴 저장

---

### Phase 3 — 데이터 분석 & 시각화

**목표:** 수동 입력 데이터를 정량적으로 분석하고 시각화한다.

#### 3-1. 히트맵
- 선수별 활동 영역 히트맵 (키프레임 위치 누적)
- 팀 전체 히트맵 (공격 점유 영역 vs 수비 점유 영역)
- 시간 구간 필터링 (전반/후반, 특정 시간대)

#### 3-2. 거리·속도 측정
- 캘리브레이션 기반 실제 거리 계산 (이미 기반 코드 있음)
- 두 선수 간 거리 실시간 표시
- 키프레임 간 이동 거리 → 추정 속도 계산
- 팀 전체 총 이동 거리 통계

#### 3-3. 패스 네트워크
- 선수 간 패스 연결선 시각화 (두께 = 패스 빈도)
- 패스 방향·성공/실패 구분
- 패스맵 오버레이 on/off

#### 3-4. 공간 분석
- 보로노이 다이어그램: 각 선수의 지배 영역 표시
- 수비 라인 높이/간격 자동 측정
- 수비 블록 컴팩트니스 지표 (선수 간 평균 거리)
- 오프사이드 라인 시각화

---

### Phase 4 — 내보내기 & 공유

**목표:** 분석 결과를 다양한 형태로 출력·공유할 수 있게 한다.

#### 4-1. 이미지/영상 내보내기
- 현재 화면 스냅샷 (PNG) — 오버레이 포함
- 미니맵 스냅샷 (SVG/PNG)
- 키프레임 구간을 GIF/MP4로 내보내기 (미니맵 애니메이션)

#### 4-2. 데이터 내보내기
- 전체 분석 세션 JSON (캘리브레이션 + 선수 위치 + 드로잉)
- CSV 내보내기 (시간, 선수, x, y 좌표 테이블)
- 통계 리포트 PDF (히트맵, 패스맵, 거리 통계 포함)

#### 4-3. 프레젠테이션 모드
- 전체 화면 모드: UI 최소화, 미니맵만 표시
- 키프레임 순차 재생 + 드로잉 오버레이 (브리핑용)
- 음성 녹음 연동 (분석 설명 첨부)

---

### Phase 5 — 자동화 & AI (장기 목표)

**목표:** 수동 작업을 줄이고 AI 기반 자동 분석으로 확장한다.

#### 5-1. 자동 선수 감지
- YOLO 등 객체 감지 모델로 파노라마 영상에서 선수 위치 자동 추출
- 캘리브레이션 데이터와 결합 → 필드 좌표 자동 매핑
- 수동 보정 UI (자동 감지 결과를 사용자가 미세 조정)

#### 5-2. 자동 캘리브레이션
- 필드 라인 자동 감지 (허프 변환 + 에지 검출)
- 감지된 라인으로 캘리브레이션 포인트 자동 생성
- 사용자 확인 후 적용

#### 5-3. 이벤트 자동 태깅
- 패스, 슈팅, 태클 등 주요 이벤트를 타임라인에 자동 마킹
- 이벤트 기반 구간 필터링 (예: "모든 슈팅 장면만 보기")

## 4. 기술 고려사항

### 아키텍처 전환 필요성
현재는 단일 HTML 파일(약 3,900줄)에 모든 로직이 들어 있다. Phase 2 이상 진행 시 다음과 같은 구조 전환을 권장한다.

- **모듈화:** ES Module 기반으로 뷰어, 캘리브레이션, 전술, 분석 모듈 분리
- **상태 관리:** 전역 변수 → 중앙 상태 스토어 (예: Zustand, 또는 커스텀 EventBus)
- **렌더링 분리:** Three.js(3D 뷰어)와 SVG(미니맵/드로잉)의 렌더 루프 독립 관리
- **데이터 영속화:** IndexedDB 또는 로컬 파일 시스템 API로 세션 자동 저장

### 성능
- 키프레임이 수백 개 이상일 때 보간 연산 최적화 (Web Worker 활용)
- 히트맵은 Canvas 2D로 별도 렌더링 (SVG보다 대량 데이터에 적합)
- 선수 감지 AI는 ONNX Runtime Web 또는 서버 사이드 추론
